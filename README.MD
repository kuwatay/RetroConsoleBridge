# RetroConsoleBridgeRetroConsoleBridge is a Wi-Fi serial console server for retro computers and single-board systems.Built on the Raspberry Pi Pico W, it bridges TCP/TELNET connections to a TTL serial console and provides a Hayes-style AT command interface for configuration and management.If you already have a [PicoWiFiModem-Retro-Edition](https://github.com/kuwatay/PicoWiFiModem-Retro-Edition) board assembled, the same hardware can be used without modification. Simply replace the firmware with RetroConsoleBridge. ## Features* Raspberry Pi Pico W based* Wi-Fi STA mode* TCP/TELNET console access* TTL serial interface* Hayes-style AT command interface* Automatic USB/Wi-Fi console switching* Configuration stored in flash memory* Optional password protection for remote access* Activity LEDs for Wi-Fi, Link, TX, and RX## System Hardware### Raspberry Pi Pico W GPIO Assignment| GPIO | Function        || ---- | --------------- || GP0  | UART TX         || GP1  | UART RX         || GP10 | Wi-Fi LED       || GP11 | Link LED        || GP12 | TX Activity LED || GP13 | RX Activity LED |### Serial InterfaceThe serial interface operates at TTL levels and is intended for systems such as:* RC2014* Z80 SBCs* 6502 SBCs* Vintage computers with TTL serial consoles## Quick StartConnect to the USB CDC console and configure Wi-Fi:```textAT$SSID=YourSSIDAT$PASS=YourPasswordAT&WATC1```Check network status:```textATIP```Connect from a modern computer:```bashtelnet <ip-address> 23```or```bashsocat -,raw,echo=0 tcp:<ip-address>:23```## AT Commands### Information```textATIATI4AT?```### Configuration```textAT&WAT&FATZAT&VAT&V1```### Wi-Fi```textAT$SSIDAT$PASSATC1ATC0ATIP```### Network```textAT$SPATTERMATPASSWD```### Console```textATCONSOLEATOATH```### Registers```textATS0ATS12ATNOTIFY```## Console Modes### AUTOThe console is normally owned by USB CDC.When a TCP/TELNET client connects:```textUSB -> Wi-Fi```When the client disconnects:```textWi-Fi -> USB```### USBConsole remains attached to USB CDC.### WIFIConsole remains attached to the network connection.## Terminal Modes### RAWRecommended for console access and file transfers.Example:```bashsocat -,raw,echo=0 tcp:<ip-address>:23```### TELNETTELNET command negotiation enabled.## Status LEDs| LED   | Description             || ----- | ----------------------- || Wi-Fi | Wi-Fi connected         || Link  | Remote client connected || TX    | UART → Network activity || RX    | Network → UART activity |## BuildingThe firmware is built using the Raspberry Pi Pico SDK.```bashmkdir buildcd buildcmake ..make```## Serial Interface and Level ConversionRetroConsoleBridge uses a 3.3V TTL serial interface.The Raspberry Pi Pico W GPIO pins are **not 5V tolerant**. Connecting a 5V serial signal directly to the Pico W RX pin may damage the hardware.### Recommended Connections#### 3.3V TTL SystemsExamples:* RC2014 with 3.3V serial interface* Modern SBCs* USB-to-TTL adapters operating at 3.3VDirect connection is possible:```textRetro System TX ---- Pico RX (GP1)Retro System RX ---- Pico TX (GP0)GND ------------ GND```#### 5V TTL SystemsA level shifter or voltage divider should be used between the retro computer TX signal and the Pico RX pin.Example:```textRetro System TX (5V)          |          +--- Level Shifter --- Pico RX (GP1)Retro System RX <------------- Pico TX (GP0)GND -------------------------- GND```The Pico TX output level of 3.3V is accepted by most TTL serial interfaces, including many RC2014 serial modules.Always verify the voltage levels of your target hardware before connecting it to the Pico W.## Client SoftwareRetroConsoleBridge can be accessed using standard TCP, TELNET, or serial terminal software.### Recommended Clients#### RAW TCP ModeRecommended for interactive console access and file transfer protocols such as XMODEM and Kermit.macOS / Linux:```bashsocat -,raw,echo=0 tcp:<ip-address>:23```Alternative:```bashnc <ip-address> 23```Note that some terminal configurations buffer keyboard input until ENTER is pressed. `socat` is recommended for true character-at-a-time operation.#### TELNET Mode```bashtelnet <ip-address> 23```Suitable for general console access.### GUI Terminal ProgramsThe following applications work well with RetroConsoleBridge:* ZOC Terminal* SyncTERM* CoolTerm* PuTTY (Windows)* Tera Term (Windows)### USB-CDC AccessThe built-in USB-CDC console can be accessed using:* screen* picocom* minicom* CoolTerm* ZOC TerminalExample:```bashscreen /dev/cu.usbmodemXXXX 115200```### File Transfer ProtocolsWhen using RAW mode, the bridge transparently passes serial data and can be used with:* XMODEM* YMODEM* KermitSupport depends on the terminal software used on the client and retro computer.## AcknowledgementsRetroConsoleBridge shares its hardware platform and
